================================================================================
          INSTRUCTIUNI DE FOLOSIRE SI CONFIGURARE - ATHLETICA AI
================================================================================

Acest proiect este o aplicatie full-stack pentru monitorizarea activitatii fizice,
a metricilor de sanatate si oferirea de sfaturi personalizate prin AI.

1. ARHITECTURA PROIECTULUI
--------------------------
*   FRONTEND-WEB: Interfata utilizator (HTML, CSS, JS) - Port: Vizualizare directa/Live Server.
*   BACKEND-JAVA: API-ul principal (Spring Boot) - Port: 8080.
*   AI-SERVICE-PYTHON: Motorul de analiza AI (FastAPI + Gemini) - Port: 8006.
*   BAZA DE DATE: MySQL (stocare utilizatori, antrenamente, metrici).

2. CERINTE PREALABILE
---------------------
*   Java 21 (JDK) si Maven instalat.
*   Python 3.10 sau mai nou.
*   MySQL Server (local sau in cloud).
*   O cheie API pentru Google Gemini (Google AI Studio).

3. CONFIGURARE SI INSTALARE
---------------------------

A. CONFIGURARE BAZA DE DATE (MySQL)
   1. Creati o baza de date numita: `fitness_app_db`.
   2. Verificati fisierul `backend-java/app/src/main/resources/application.properties`.
   3. Daca doriti sa folositi baza de date locala, modificati:
      spring.datasource.url=jdbc:mysql://localhost:3306/fitness_app_db
      spring.datasource.username=UTILIZATORUL_TAU
      spring.datasource.password=PAROLA_TA

B. CONFIGURARE AI SERVICE (Python)
   1. Navigati in folderul `ai-service-python/`.
   2. Creati un fisier `.env` si adaugati cheia API:
      GEMINI_API_KEY=cheia_ta_aici
   3. Instalati dependentele:
      pip install -r requirements.txt
   4. Porniti serviciul:
      python main.py
   *Nota: Serviciul va rula pe http://127.0.0.1:8006.*

C. CONFIGURARE BACKEND (Java)
   1. Navigati in folderul `backend-java/app/`.
   2. Compilati proiectul:
      mvn clean install
   3. Porniti aplicatia:
      mvn spring-boot:run
   *Nota: Backend-ul va rula pe http://127.0.0.1:8080.*

D. PORNIRE FRONTEND
   1. Navigati in folderul `frontend-web/`.
   2. Deschideti `auth.html` in browser (recomandat folosind "Live Server" din VS Code).

4. CUM FUNCTIONEAZA APLICATIA
-----------------------------

1. AUTENTIFICARE:
   - Prima data, accesati `auth.html` pentru inregistrare sau logare.
   - Dupa logare, datele utilizatorului sunt salvate in LocalStorage.

2. DASHBOARD:
   - Aici puteti vedea un sumar al activitatii.
   - Butonul "Scaneaza din nou" trimite datele tale (antrenamente recente + metrici) catre Backend, care apoi apeleaza Serviciul AI (Python) pentru a genera sfaturi personalizate.

3. ANTRENAMENTE:
   - Puteti adauga antrenamente detaliate (tip, durata, intensitate, puls, calorii).
   - Acestea sunt salvate in MySQL si folosite pentru analiza istoricului.

4. METRICI SANATATE:
   - Puteti introduce zilnic date precum HRV (Heart Rate Variability), ore de somn, nivel de stres si puls de repaus. Acestea sunt esentiale pentru ca AI-ul sa determine nivelul de recuperare.

5. NUTRITIE (AI Analysis):
   - Introduceti text liber (ex: "Am mancat o salata cu pui si avocado").
   - AI-ul va estima caloriile, macronutrientii si va oferi feedback in contextul obiectivului tau de fitness.

6. RECUPERARE:
   - Raportati zonele cu febra musculara sau dureri.
   - AI-ul va genera un protocol de recuperare (stretching, odihna, etc.) si timpul estimat pana la refacerea completa.

5. DEPANARE (Troubleshooting)
-----------------------------
*   ERORI DE CONEXIUNE: Asigurati-va ca backend-ul Java (8080) si serviciul Python (8006) ruleaza simultan.
*   AI NU RASPUNDE: Verificati daca GEMINI_API_KEY este valida. Daca nu este setata, serviciul Python va returna date de tip "Demo" pentru a permite testarea interfetei.
*   CORS ISSUES: Backend-ul este configurat sa accepte cereri de la originile standard, dar asigurati-va ca browserul nu blocheaza cererile catre 127.0.0.1.
